Introducción¶

El objetivo principal de un modelo entrenado es que tenga la capacidad de generalizar, es decir, la capacidad de predecir bien nuevos datos que no formen parte de la muestra de entrenamiento (por ejemplo, una muestra de datos de test). La relacin entre el nmero de observaciones y el nmero de variables independientes es clave en la capacidad de generalizacin de los modelos.

Para evitar el sobreajuste en situaciones donde el nmero de observaciones es del orden, o menor, que el de grados de libertad del sistema y/o donde hay colinealidad, se utilizan (entre otras) tcnicas como la seleccin de variables y la regularizacin. La primera permite seleccionar de forma objetiva un subconjunto de variables predictoras, mientras que la segunda acta sobre los parmetros del modelo, impidiendo que tomen valores elevados que haran al modelo muy sensible a diferentes muestras de entrenamiento.

Base de datos: Breast cancer¶

En este ejemplo trabajaremos con la base de datos Breast cancer (Diagnostic), incluida en Kaggle. El objetivo de este conjunto de datos es clasificar a las pacientes en base a si el cncer de mama que tienen es benigno (B) o maligno (M). Para ello se utilizan 31 predictores relacionados con las propiedades de las clulas cancergenas, que en comparacin con el nmero de observaciones (569) es bajo.

Selección de variables¶

Selección del mejor subconjunto¶

La funcin regsubsets (?regsubsets) de la librera leaps nos permite realizar la seleccin del mejor subconjunto con un nmero dado de predictores. La sintaxis es similar a la usada para el modelo lineal (lm) y, del mismo modo, el comando summary nos permite explorar los resultados. Utilizamos el argumento nvmax para forzar la utilizacin de todas las variables (30), ya que por defecto la funcin regsubsets solo utiliza las 8 primeras.

In [43]:
#install.packages('leaps')
library(leaps)
regfit.full <- regsubsets(diagnosis ~., df, nvmax = 30)
full.summary <- summary(regfit.full)
names(full.summary)

  2. 'rsq'

La funcin summary devuelve medidas del ajuste que se pueden usar para seleccionar el mejor modelo. En particular, si revisamos los valores del ajuste, en este caso rsq, se observa que el estadstico va mejorando a medida que aumentamos el nmero de variables en el modelo. Sin embargo, si nos fijamos en el estadstico BIC (Bayesian information criterion), que penaliza la complejidad del modelo, se observa que los valores ptimos se encuentran considerando 11 variables.

Cada fila de este grfico representa el mejor modelo ajustado para un nmero dado de variables. Cada cuadrado de color representa que la variable entra en ese modelo, mientras que la intensidad del color se asocia con la escala del eje Y. initialModel_0 <- glm(diagnosis ~ 1, data = df, family=binomial(link="logit"))
initialModel_p <- glm(diagnosis ~ ., data = df, family=binomial(link="logit"))

fw <- step(initialModel_0,
  scope = list(lower = formula(initialModel_0), upper = formula(initialModel_p)),
  direction = "forward"
)
summary(fw)

Regularización¶

La regularizacin consiste en aadir un trmino a la funcin objetivo que elimine predictores (regularizacin L1) o disminuya notablemente su influencia (regularizacin L2) de tal manera que su capacidad de generalizacin mejore. El coeficiente estimado estar sesgado, ya que se ha aadido un trmino a la funcin objetivo, sin embargo su varianza puede disminuir. Este equilibrio entre el sesgo y la varianza es lo que se conoce como bias-variance tradeoff. Si el modelo est demasiado sesgado entonces no se capturarn correctamente las relaciones entre predictando y predictor, sin embargo si est muy poco sesgado, la varianza del coeficiente estimado puede ser alta y no generalizar bien.

En R, para regularizar modelos lineales generalizados se utiliza el paquete glmnet. El tipo de regularizacin viene dado por el valor del parmetro alpha: para una regularizacin L1, alpha = 1, para una regularizacin L2, alpha = 0. Ambos tipos de regularizacin penalizan la norma (1 2, en funcin de si la regularizacin es L1 o L2) de los coeficientes estimados. Una consecuencia de la regularizacin L1 es que indirectamente selecciona variables al hacer estrictamente 0 algunos coeficientes. El nmero de coeficientes que se hacen 0 viene determinado por el valor de lambda. El valor de lambda ptimo se obtiene mediante validacin cruzada a travs de la funcin cv.glmnet.

En primer lugar, debemos instalar y cargar las libreras que utilizaremos en los ejemplos:
